User Portal

This project established a centralized hub where users could access information, communicate, request support, and perform key job-related tasks.

  • Role: UX Lead
  • Focus: Research, Design, development and testing.
  • Timeline: 12 weeks
Overview Problem Process Solution Impact Reflection

Overview

This project focused on designing a front-facing static user portal for engineers, analysts, program managers, and leadership. Its primary objective was to consolidate information, documentation, communication, resources, and updates into a single, accessible location. The effort supported key organizational goals by improving operational efficiency, reducing support overhead, and establishing a dependable source of truth across teams. I identified the core issues by uncovering communication gaps, outdated resources, and a lack of clear points of contact for various requests. Through user conversations and research, I mapped out chokepoints and defined a path toward a unified solution. The portal quickly became a widely used destination for accurate information, Tier 1 helpdesk support, and streamlined communication. The project operated under significant technical and resource constraints, as I was responsible for the full UX lifecycle‚Äîincluding research, wireframing, design, testing, and front-end development. Delivered within a 12-week window, the work involved close collaboration with various users and stakeholders, while cross-functional partners ensured compliance and security requirements were met. Despite the limitations, the portal provided measurable value and improved clarity, accessibility, and operational flow across the organization.
